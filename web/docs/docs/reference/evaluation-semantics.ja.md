# 評価セマンティクス

このページは、PalmScript の式と文がランタイムでどのように評価されるかを定義します。

## 実行モデル

PalmScript はスクリプトを一度コンパイルし、ベースクロックの各ステップで一度評価します。

各ステップで:

1. ランタイムが、そのステップの現在の市場シリーズサンプルを具体化する
2. より遅いインターバルのフィードは、その時点までにローソク足が完全に確定している場合にのみ進む
3. bytecode program が実行される
4. そのステップの `plot`、`export`、`trigger` 出力が収集される

market mode のフィードごとにステップ入力の構築方法は異なり得ますが、ステップ開始後の式評価は同じです。

## 式カテゴリ

式は、スカラー値またはシリーズ値の現在サンプルに評価されます。

シリーズ式では:

- 一つのステップにおける式結果は一つの現在サンプル
- インデックスは、その式自身の更新クロック上の過去サンプルを参照する

## 演算子優先順位

PalmScript は次の順序で演算子を評価します。低いものから高いものへ:

1. `or`
2. `and`
3. `==`, `!=`, `<`, `<=`, `>`, `>=`
4. `+`, `-`
5. `*`, `/`
6. 単項 `-`, 単項 `!`
7. 呼び出し、インデックス、修飾 postfix

同じ優先順位の演算子は左結合です。

## 算術

算術演算子は `+`, `-`, `*`, `/` です。

ルール:

- 両オペランドは数値、数値 series、または `na` でなければならない
- いずれかのオペランドが `na` なら結果は `na`
- いずれかのオペランドが `series<float>` なら結果は `series<float>`
- そうでなければ結果は `float`

## 比較

比較演算子は `==`, `!=`, `<`, `<=`, `>`, `>=` です。

ルール:

- `<`, `<=`, `>`, `>=` は数値オペランドを必要とする
- `==` と `!=` は `na` でない任意のオペランドに定義される
- 異種型同士の equality は不一致として比較される
- いずれかのオペランドが `na` なら結果は `na`
- いずれかのオペランドがシリーズなら結果は series boolean
- そうでなければ結果は `bool`

## 単項演算子

PalmScript は次をサポートします。

- 数値オペランドに対する単項 `-`
- 真偽値オペランドに対する単項 `!`

ルール:

- 単項演算子は `na` を伝播する
- `series<float>` に対する単項 `-` は `series<float>` を返す
- `series<bool>` に対する単項 `!` は `series<bool>` を返す

## 論理演算子

`and` と `or` は `bool`、`series<bool>`、または `na` を必要とします。

これらは決定的な三値論理を使います。

### `and`

| Left | Right | Result |
| --- | --- | --- |
| `false` | `false` | `false` |
| `false` | `true` | `false` |
| `false` | `na` | `false` |
| `true` | `false` | `false` |
| `true` | `true` | `true` |
| `true` | `na` | `na` |
| `na` | `false` | `false` |
| `na` | `true` | `na` |
| `na` | `na` | `na` |

### `or`

| Left | Right | Result |
| --- | --- | --- |
| `true` | `true` | `true` |
| `true` | `false` | `true` |
| `true` | `na` | `true` |
| `false` | `true` | `true` |
| `false` | `false` | `false` |
| `false` | `na` | `na` |
| `na` | `true` | `true` |
| `na` | `false` | `na` |
| `na` | `na` | `na` |

PalmScript は論理演算子を適用する前に両オペランドを評価します。言語は短絡評価を保証しないため、論理式は通常の言語ルールの中で eager に解析・実行されます。

## `if` セマンティクス

`if` は式形式ではなく文形式です。

ルール:

- 条件は `bool`、`series<bool>`、または `na` に評価されなければならない
- `if` 条件中の `na` は分岐選択では false として扱われる
- 各ステップで必ず一方の分岐だけが実行される
- `else` が必須であるため、両分岐は構文的に存在しなければならない

## 関数評価

ユーザー定義関数は式本体であり、ランタイム動的 dispatch ではなく特殊化によってコンパイルされます。

ルール:

- 引数数は宣言パラメータ数と一致しなければならない
- 関数は引数型と更新クロックで特殊化される
- 再帰および循環関数グラフはコンパイル時に拒否される
- 関数本体は `plot` を呼べない

## ノールックアヘッド規則

PalmScript は、未確定の上位インターバルローソク足を公開してはなりません。

結果:

- 上位インターバル series は、そのローソク足が完全に確定した後でのみ変化する
- 上位インターバル series 上のインデックスは、完全に確定した上位インターバルサンプルの履歴をたどる
- ソース対応の補助インターバルも同じ規則に従う

## Builtin ヘルパーのセマンティクス

builtin ヘルパーの計算式、インジケーター契約、window ルール、`na` 挙動は [Builtins](builtins.md) と [インジケーター](indicators.md) セクションで定義されます。

ルール:

- ヘルパー builtin は、その series 入力の更新クロックに従う
- ヘルパー出力は、このページで定義された同じ値規則と `na` 規則を通じて `if`、インデックス、さらなる builtin 呼び出しに参加する
- 条件が `crossover(...)` のようなヘルパーから来た場合でも、`if` は分岐選択で `na` を false として扱う

## 決定性

式評価は決定的です。

戦略実行中、言語セマンティクスは次にのみ依存します。

- コンパイル済みプログラム
- 準備済み入力フィード
- 設定された VM 制限

wall-clock time、filesystem access、randomness、network access には依存しません。
