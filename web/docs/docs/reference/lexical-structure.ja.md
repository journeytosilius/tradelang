# 字句構造

PalmScript ソースは構文解析の前にトークン化されます。レキサはソース順序とスパンを保持し、不明な文字と無効なインターバルリテラルを拒否しなければなりません。

## キーワード

次のトークンは予約キーワードです。

- `fn`
- `let`
- `interval`
- `source`
- `use`
- `const`
- `input`
- `export`
- `regime`
- `trigger`
- `cooldown`
- `max_bars_in_trade`
- `entry`
- `exit`
- `protect`
- `target`
- `order`
- `if`
- `else`
- `and`
- `or`
- `true`
- `false`
- `na`

これらのキーワードは、識別子が必要な場所では使えません。

## 識別子

識別子は次の条件を満たします。

- ASCII の英字または `_` で始まる
- 以降は ASCII の英字、数字、または `_` を含められる

例:

- `trend`
- `_tmp1`
- `weekly_basis`

## リテラル

### 数値リテラル

数値リテラルは `f64` として解析されます。

受理される形式:

- `1`
- `14`
- `3.5`

拒否される形式:

- `1e6` のような指数表記
- `.5` のような先頭ドット形式

負数は独立した符号付きリテラルトークンではなく、単項 `-` によって表現されます。

### 真偽値リテラル

真偽値リテラルは次の二つです。

- `true`
- `false`

### 欠損値リテラル

`na` は欠損値リテラルです。

### 文字列リテラル

文字列リテラルは現在、文法が `source` 宣言内で許可する場所でのみ受理されます。

文字列は:

- `"` で囲まれる
- `"`, `\\`, `\\n`, `\\r`, `\\t` の基本エスケープを含められる
- エスケープされていない改行をまたいではならない

## コメント

サポートされるのは単一行コメントだけです。

```palmscript
// trend regime
let fast = ema(spot.close, 5)
```

単独の `/` トークンは算術除算演算子です。`//` は単一行コメントを開始します。

## 文の区切り

文は次で区切られます。

- 改行
- セミコロン

括弧や角括弧の中の改行は文を終了させません。

## インターバルリテラル

インターバルリテラルは大文字小文字を区別します。受理される集合は [インターバル一覧](intervals.md) で定義されます。

例:

- `1w` は有効
- `1M` は有効
- `1W` は無効

## `optimize` について

`optimize` は新しい予約語です。`input ... optimize(...)` のメタデータ接尾辞として使われるため、通常の識別子としては再利用できません。
