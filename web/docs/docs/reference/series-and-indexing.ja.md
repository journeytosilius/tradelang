# シリーズとインデックス

シリーズ値は、履歴が有界な時間インデックス付きサンプルを表します。

## 市場シリーズの形式

PalmScript は市場シリーズをソース修飾形式でのみ公開します。

```palmscript
bn.close
hl.1h.close
```

ルール:

- `<alias>.<field>` は、そのソースのスクリプトベースインターバルを参照する
- `<alias>.<interval>.<field>` は、そのソースの指定インターバルを参照する
- `close` のような裸の市場識別子は拒否される
- 上位ソースインターバル参照には対応する `use <alias> <interval>` 宣言が必要

## 現在サンプルのセマンティクス

シリーズがインデックスなしで使われると、その式は現在の実行ステップにおけるそのシリーズの現在サンプルとして評価されます。

## インデックス

インデックスの形式:

```palmscript
x[n]
```

ルール:

- `n` は非負の整数リテラルでなければならない
- 動的インデックスは拒否される
- インデックスできるのはシリーズ値だけ
- `x[0]` は現在サンプルを参照する
- `x[1]` は前のサンプルを参照する
- `x[n]` は、そのシリーズ自身の更新クロックで `n` 更新前のサンプルを参照する

十分な履歴がない場合、インデックス式は `na` に評価されます。

## 更新クロックの所有

各シリーズは自分自身の更新クロックで進みます。

例:

- `bn.close[1]` はベースインターバルに従う
- `hl.1h.close[1]` は 1 時間クロック上のソース `hl` に従う

派生シリーズは入力の更新クロックを継承します。遅いシリーズは、進んでいない高速クロック上で再カウントされません。

## 欠損サンプル

シリーズが現在サンプルで `na` を返すのは次のような場合です。

- インデックスに必要な履歴が不足している
- 宣言済みソースのベースタイムスタンプの和集合で作られたベースクロックステップに、ソースフィードが存在しない
- シリーズがまだ一度も確定していない上位インターバルフィードである
- インジケーターがまだウォームアップ中である

## 時刻シリーズ

`time` は、UTC の Unix ミリ秒で表したローソク足始値時刻をサンプルとする数値シリーズです。

ルール:

- ベース `time` はベースインターバルのローソク足始値時刻を公開する
- 上位インターバル `time` はその上位インターバルのローソク足始値時刻を公開する
- ソース修飾された `time` は、価格や出来高フィールドと同じソース / インターバル選択規則に従う
