# 診断

PalmScript は三つの公開レイヤーから診断とエラーを返します。

## 1. コンパイル診断

コンパイル診断は、スパンを持つソースレベルの失敗です。

診断クラス:

- 字句エラー
- 構文解析エラー
- 型エラーと名前解決エラー
- コンパイル時の構造エラー

例:

- `interval` の欠落または重複
- 未対応の `source` テンプレート
- 不明な source エイリアス
- 未宣言の `use` インターバル参照
- ベースより低いインターバル参照
- 重複した束縛
- 無効な関数再帰
- 無効な builtin の arity または引数型

これらの診断は次を通じて表示されます。

- ブラウザ IDE のエディタ診断パネル
- ホストアプリが発行するバックテスト要求

## 2. 市場データ取得エラー

コンパイル成功後、必要な履歴フィードの組み立て中にランタイム準備が失敗することがあります。

例:

- 要求した時間ウィンドウが無効
- スクリプトに `source` 宣言がない
- 取引所リクエストが失敗する
- venue 応答が不正
- 必要なフィードが要求ウィンドウ内でデータを返さない
- シンボルを選択した venue が解決できない

## 3. ランタイムエラー

ランタイムエラーは、フィード準備開始後または実行中に発生します。

例:

- フィード整列エラー
- ランタイムフィードの欠落または重複
- 命令予算の使い尽くし
- スタックアンダーフロー
- 実行中の型不一致
- 無効なローカルまたはシリーズスロット
- 履歴容量オーバーフロー
- 出力収集中の出力型不一致

## レイヤーの責務

失敗をどのレイヤーが所有するかは契約の一部です。

- 構文と意味の妥当性はコンパイルの責務
- 取引所 / ネットワーク / 応答の妥当性は市場データ取得の責務
- フィード整合性と実行の妥当性はランタイムの責務

PalmScript は意味を黙って劣化させるのではなく、明示的に失敗します。
